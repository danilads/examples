// ------------------------------
// --- [React
// ------------------------------
JS-библиотека для построения UI, основана на компонентном подходе.
Virtual DOM для оптимизации работы с реальным DOM


// ------------------------------
// --- [Virtual DOM]
// ------------------------------
 (VDOM) — это лёгкая копия реального DOM, представленная в виде JavaScript-объектов.

// ------------------------------
// --- [Основные концепции]
// ------------------------------

Компоненты (функциональные и классовые). (Переиспользование компонетов)
JSX — синтаксис, похожий на HTML, но под капотом — React.createElement.
Пропсы (props) — входные данные в компонент.
Состояние (state) — внутренняя "память" компонента.


// ------------------------------
// --- [Рендеринг и оптимизация]
// ------------------------------

Virtual DOM — минимизация обновлений реального DOM.
Ключи (key) — помогают React отслеживать элементы списка.
Мемоизация:
  useMemo и useCallback / React.memo


// ------------------------------
// --- [Контролируемые vs Неконтролируемые компоненты]
// ------------------------------

в Контролируемым - формы управляется React через state

// ------------------------------
// --- [Ре-Рендер компонента]
// ------------------------------

1) изменение props
2) изменение state
3) ре-рендер родителя
4) обновления context / redux

// ------------------------------
// --- [Батчинг]
// ------------------------------

Батчинг (batching) — это оптимизация, при которой React объединяет несколько вызовов setState в один ререндер, чтобы не перерисовывать компонент слишком часто.


// ------------------------------
// --- [Reconciliation]
// ------------------------------

Reconciliation — это алгоритм React, который решает, как именно обновить DOM, когда состояние или пропсы компонента меняются.

Проще говоря:
У тебя есть старое дерево элементов (Virtual DOM).
React получает новое дерево (после setState или изменения пропсов).
React сравнивает старое и новое дерево, чтобы понять:

что можно оставить как есть,
что нужно изменить,
что нужно удалить или добавить.

Этот процесс называется согласованием (reconciliation).
