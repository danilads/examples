1) Парсинг HTML → DOM

Браузер читает HTML и строит DOM-дерево (Document Object Model).

2) Парсинг CSS → CSSOM

Браузер загружает и разбирает CSS, формируя CSSOM-дерево (CSS Object Model).

3) Создание Render Tree

DOM и CSSOM объединяются в Render Tree, где есть только видимые элементы (например, скрытые через display: none — не включаются).

4) Layout (Reflow)

Рассчитывается геометрия элементов: размеры, позиции, отступы и т. д.

5) Painting (отрисовка)

Браузер рисует пиксели на экране согласно дереву рендеринга.

1) скрипты могут блокировать рендеринг
Решения:
Использовать async для скриптов, которые не зависят от DOM.
Использовать defer для скриптов, которые должны выполняться после парсинга HTML.

2) Web Fonts
Если шрифт загружается через @font-face, браузер может ждать его загрузки перед рендерингом текста (FOIT – Flash of Invisible Text).
Решения: font-display: swap; для отображения fallback-шрифта сразу.

проходит несколько этапов обработки

HTML → DOM
CSS → CSSOM
DOM + CSSOM → Render Tree
Render Tree → Layout → Paint → Composite → Screen

1) Парсинг HTML → построение DOM

Читает HTML построчно.

Превращает теги в DOM-дерево (Document Object Model).

Если встречает <script> без defer или async

2) Загрузка и обработка CSS

Когда браузер обнаруживает <link rel="stylesheet"> или <style>:

➜ Строит CSSOM (CSS Object Model)

Загружает CSS-файлы.

Парсит правила, селекторы и значения.

Создаёт CSSOM — структуру, описывающую, какие стили применяются куда.

⚠️ Пока CSS не загружен, рендеринг блокируется (браузер не покажет страницу полностью).

3)  Выполнение JavaScript

Когда доходит до JS-файлов или <script>:

Загружает скрипт.

Выполняет его в JavaScript-движке (V8, SpiderMonkey и др.).

JS может менять DOM и стили.

Если скрипт указан как:

async → загружается параллельно и выполняется сразу после загрузки

defer → загружается параллельно, выполняется после построения DOM

обычный → блокирует HTML-парсер

4) Создание Render Tree

Браузер объединяет DOM + CSSOM 👉 Render Tree
Это дерево отображаемых элементов — скрытые (display:none) не попадают туда.

5) . Layout (расчёт геометрии)

Браузер рассчитывает:

размер элементов,

их положение,

модель коробок (box model),

поток документа.

Это называется layout или reflow.


6) Painting (рисование) и Композиция (compositing)
На основе Render Tree браузер рисует:

текст

фоны

границы

изображения

тени
И кладёт всё в слои. 

Графический движок объединяет слои в итоговую картинку и выводит её на экран.

// ------------------------------
// --- [ defer или async ]
// ------------------------------

<script defer>
✔️ Загружается параллельно с HTML
✔️ Не блокирует разбор HTML
✔️ Гарантирует порядок выполнения
✔️ Выполняется после построения DOM, но до события DOMContentLoaded


<script async>
✔️ Загружается параллельно
❌ Выполняется сразу после загрузки, даже если HTML ещё парсится
❌ Порядок не сохраняется
❌ Может «вклиниться» в разбор HTML
